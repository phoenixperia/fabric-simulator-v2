# Mask Creation Workflow
# FIERO Fabric Simulator V2

---

# Overview

This document defines the official mask production workflow used throughout FIERO Fabric Simulator V2.

The purpose of this workflow is to create:

```txt
stable reusable production masks
```

through a controlled step-by-step process.

The workflow is designed to preserve:

- silhouette continuity
- overlap stability
- projection compatibility
- rendering consistency
- future scalability

---

# Core Philosophy

Mask creation is NOT image editing.

Mask creation is:

```txt
structural renderer construction
```

inside the V2 architecture.

---

# Primary Goals

The workflow must preserve:

- clean geometry
- reusable structure
- stable hierarchy
- luxury tailoring realism
- modular rendering behavior

---

# Important Philosophy

The renderer prioritizes:

```txt
structural stability
```

over:

```txt
speed of production
```

---

# Official Workflow Order

The official V2 mask workflow is:

```txt
1. Master Mannequin Fix
2. Blueprint Definition
3. Structural Mask Creation
4. Overlap Validation
5. Projection Validation
6. Shadow Validation
7. Outline Validation
8. Render Test
9. Cleanup
10. Production Approval
```

---

# Step 1 — Master Mannequin Fix

## Purpose

Establish immutable rendering foundation.

---

# Required Assets

```txt
master_front_v001.png
master_45_v001.png
master_side_v001.png
master_back_v001.png
```

---

# Important Rule

The mannequin must remain fixed before mask creation begins.

Changing:

- pose
- shoulder position
- camera
- lighting
- proportions

invalidates mask alignment.

---

# Step 2 — Blueprint Definition

## Purpose

Define garment structure before mask production.

---

# Required Documents

```txt
single_open_blueprint.md
lapel_rules.md
overlap_rules.md
```

---

# Important Rule

Masks should never be created before blueprint structure is finalized.

---

# Step 3 — Structural Mask Creation

## Purpose

Create silhouette-defining regions.

---

# Core Regions

```txt
front_panel
lapel
sleeve
collar
v_zone
hem
side_body
```

---

# Important Rule

Structural masks define renderer stability.

This stage has extremely high importance.

---

# Step 4 — Overlap Validation

## Purpose

Validate layer relationships.

---

# Validation Targets

```txt
tie overlap
lapel hierarchy
collar hierarchy
sleeve continuity
coat compatibility
```

---

# Important Rule

All layers must remain independently readable.

---

# Step 5 — Projection Validation

## Purpose

Validate fabric behavior.

---

# Validation Targets

```txt
pinstripe continuity
lapel alignment
sleeve flow
seam continuity
```

---

# Important Rule

Projection realism must preserve silhouette readability.

---

# Step 6 — Shadow Validation

## Purpose

Validate depth realism.

---

# Validation Targets

```txt
lapel shadows
fold shadows
overlap shadows
sleeve depth
```

---

# Important Rule

Shadows should remain:

- soft
- elegant
- controllable

---

# Step 7 — Outline Validation

## Purpose

Validate silhouette readability.

---

# Validation Targets

```txt
lapel edge
shoulder flow
hem continuity
sleeve edge
```

---

# Important Rule

Outlines must remain subtle.

Avoid graphic edge appearance.

---

# Step 8 — Render Test

## Purpose

Validate full renderer behavior.

---

# Required Tests

```txt
solid fabric
pinstripe
glen check
shadow stability
overlap stability
```

---

# Important Rule

Render testing must occur before production approval.

---

# Step 9 — Cleanup

## Purpose

Perform refinement and stabilization.

---

# Cleanup Targets

```txt
edge cleanup
artifact removal
projection correction
shadow balancing
```

---

# Important Rule

Cleanup should remain minimal.

Avoid excessive correction dependency.

---

# Step 10 — Production Approval

## Purpose

Approve stable renderer state.

---

# Approval Requirements

```txt
stable silhouette
stable overlap
stable projection
stable shadows
stable outlines
```

---

# Important Rule

Only stable systems should enter production.

# Validation Philosophy

```txt
renderer stability
```

---

# Collision Prevention Goals

The workflow must prevent:

- lapel clipping
- collar collapse
- tie corruption
- sleeve instability
- silhouette collapse

---

# Rendering Quality Standard

Mask production should achieve:

```txt
stable luxury tailoring realism
```

before pursuing advanced visual experimentation.

---

# Explicit Non-Goals

This workflow does NOT prioritize:

- rapid mask generation
- artistic stylization
- cinematic effects
- exaggerated fashion rendering

---

# Long-Term Compatibility

This workflow is designed to support future expansion into:

- ladies systems
- outerwear systems
- oversized tailoring
- layered winterwear
- advanced configurators
- automated rendering pipelines

without rebuilding production architecture.

---

# Final Principle

Mask creation defines renderer stability.

Luxury tailoring presentation depends heavily on:

- structured workflow
- stable geometry
- readable overlap
- controlled projection
- disciplined production behavior
